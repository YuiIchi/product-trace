name: product-trace-new-requirement
description: >-
  新需求门控。当Sprint进行中有新需求进来、不属于当前已规划Story范围时，MUST 使用此SKILL。
  典型触发词："加个XX功能"、"来个紧急需求"、"线上出问题了"、"这个能不能顺便做了"。
  你将判定：增量(加入Backlog)→新Feature(独立走Discover)→紧急(插入当前Sprint,标记[!],必须记录代价)。
  不让新需求无声溜进Sprint。若需求范围很小且明确属于已有Story——直接在spec的Task里加，不需要走这个流程。
  关键词：新需求、增量、紧急、Feature、Backlog、门控。
