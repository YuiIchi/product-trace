description: >-
  开发中纠偏。当实现过程中发现之前的设计(spec/ROADMAP/vision)与实际不一致时，MUST 使用此SKILL。
  典型触发词："等等这个不对"、"应该是..."、"设计有问题"、"我改主意了"。
  你将判定偏差等级：L0(技术细节,30秒)→L1(验收标准,完成Task后)→L2(功能范围,停手)→L3(产品方向,回Discover)。
  每次纠偏留CORR记录、按等级上行更新受影响的文档。不是代码重构——是设计层面的偏差。
  关键词：纠偏、CORR、SIGNAL、L0、L1、L2、L3、设计变更、漂移。
